## 2026-04-19 긴급 이슈 분석 + Kakao 코드-온리 해결

### 🔴 확인된 문제점 (분류)

#### 1. iOS 신규 가입자 Kakao 인증 실패 — **가장 시급**
- 에러: `NSURLErrorDomain Code: -1002 지원되지 않는 URL`
- 원인: Kakao 웹 인증이 `kakaotalk://` 스킴으로 카톡 앱 전환 시도 → WKWebView는 http/https만 처리
- 영향: 신규 가입자 인증 완전 불가 → 앱 사용 자체 불가
- 기존 가입자는 Kakao SSO 쿠키 잔존으로 앱 전환 없이 성공 (부작용)

#### 2. iOS 로그아웃 후 자동 로그인 — **부차적 이슈**
- 현상: 로그아웃 후 Kakao/Google 클릭 시 재인증 창 없이 바로 로그인
- 원인: WKWebView `sharedCookies=true` → Safari 저장소의 SSO 쿠키 잔존
- 비교: Android TWA는 Chrome Custom Tabs 쿠키 격리로 정상 (재인증 창 뜸)

#### 3. Apple Face ID 매번 요구 — **정상 동작 확인**
- 이유: Apple Privacy by Design (쿠키 SSO 없음, JWT+생체인증)
- 결론: 모든 앱 동일, 수정 불필요

#### 4. Google 한국어 오류 — **원인 불명**
- 실제 에러 메시지 미확보 → 추가 정보 필요
- Build 7 Safari UA 위장으로 현재 대부분 동작
- 코드-온리 근본 해결 불가 (Google 정책상 WebView OAuth 금지, 2021-09 완전 차단)

### ✅ 해결한 것

#### Kakao `prompt=login` 파라미터 추가

- **영향 범위**: Google/Apple/로그아웃 모두 안 건드림
- **재빌드**: 불필요 (서버만, Replit 배포)

## 2026-04-19 (추가) Kakao 로그인 두 버튼 차이 분석 + 상단 버튼 복구 계획

### 🔍 두 버튼 정확한 차이 (공식 문서 기반)

| 항목 | 상단 "카카오톡으로 로그인" | 하단 "계정 정보 입력" |
|------|---------------------------|----------------------|
| 스킴 | `kakaokompassauth://` (custom) | `https://kauth.kakao.com/...` |
| 흐름 | 웹 → 카톡 앱 전환 → 앱 내 인증 → `kakao{APP_KEY}://` 콜백 | 웹 폼 입력 → HTTPS OAuth 리다이렉트 |
| UX | 1터치 (사용자 선호 ✅) | 계정/비번 수동 입력 |
| iOS 결과 | ❌ -1002 (WKWebView가 custom scheme 차단) | ✅ 성공 |
| 백엔드 로직 | **동일** (둘 다 authorization code → access token) | **동일** |

**핵심**: 백엔드 차이 없음. 차이는 **인증 프론트엔드**(앱 SSO vs 웹 폼).

### ⚠️ prompt=login 재평가 (투명 보고)

- Kakao 공식 문서 재확인: `prompt=login`은 **"재인증 강제"** 기능
- 이전 대화에서 "앱 전환 차단" 효과라고 적용했지만 **공식 근거 없는 추정**에 가까웠음 (Ralph Roof 검증 부족 사례)
- 실제 효과는 iPhone 실기기 테스트로만 확정 가능
- 만약 실제로 앱 전환을 막는다면 **사용자 선호(상단 버튼 1터치)와 정반대 효과**

### 🎯 대대적 개편 시 상단 버튼 복구 계획 (옵션 C)

공식 요구사항 4가지 (`@react-native-kakao` v2.4.5, Kakao iOS 문서):

1. **`mobile-app/app.json` ios.infoPlist** 추가:
   ```json
   "LSApplicationQueriesSchemes": ["kakaokompassauth", "kakaolink"],
   "CFBundleURLTypes": [
     { "CFBundleURLSchemes": ["kakao{NATIVE_APP_KEY}"] }
   ]
   ```

2. **`mobile-app/App.js` WebView** `onShouldStartLoadWithRequest` 추가:
   - URL이 `kakaokompassauth://` 또는 `kakaotalk://`로 시작 → `Linking.openURL(url)` + `return false`
   - https/기타는 그대로 통과

3. **딥링크 핸들러**: `kakao{APP_KEY}://` 수신 시 WebView에 인증 코드 주입 (Build 7 `handleDeepLink` 인프라 재활용 가능)

4. **`prompt=login` 동시 롤백**: 상단 버튼 작동시키려면 앱 전환 허용해야 함 → `server/kakaoAuth.ts:105` 제거
