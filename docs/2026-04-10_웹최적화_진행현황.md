## 2026-04-19 긴급 이슈 분석 + Kakao 코드-온리 해결

### 🔴 확인된 문제점 (분류)

#### 1. iOS 신규 가입자 Kakao 인증 실패 — **가장 시급**
- 에러: `NSURLErrorDomain Code: -1002 지원되지 않는 URL`
- 원인: Kakao 웹 인증이 `kakaotalk://` 스킴으로 카톡 앱 전환 시도 → WKWebView는 http/https만 처리
- 영향: 신규 가입자 인증 완전 불가 → 앱 사용 자체 불가
- 기존 가입자는 Kakao SSO 쿠키 잔존으로 앱 전환 없이 성공 (부작용)

#### 2. iOS 로그아웃 후 자동 로그인 — **부차적 이슈**
- 현상: 로그아웃 후 Kakao/Google 클릭 시 재인증 창 없이 바로 로그인
- 원인: WKWebView `sharedCookies=true` → Safari 저장소의 SSO 쿠키 잔존
- 비교: Android TWA는 Chrome Custom Tabs 쿠키 격리로 정상 (재인증 창 뜸)

#### 3. Apple Face ID 매번 요구 — **정상 동작 확인**
- 이유: Apple Privacy by Design (쿠키 SSO 없음, JWT+생체인증)
- 결론: 모든 앱 동일, 수정 불필요

#### 4. Google 한국어 오류 — **원인 불명**
- 실제 에러 메시지 미확보 → 추가 정보 필요
- Build 7 Safari UA 위장으로 현재 대부분 동작
- 코드-온리 근본 해결 불가 (Google 정책상 WebView OAuth 금지, 2021-09 완전 차단)

### ✅ 해결한 것

#### Kakao `prompt=login` 파라미터 추가
- **파일**: [server/kakaoAuth.ts:55-107](server/kakaoAuth.ts#L55-L107) (7줄 추가)
- **방식**: `authorizationParams()` 오버라이드로 authorize URL에 `prompt=login` 자동 추가
- **근거**: Kakao REST API 공식 문서 — *"prevents automatic app switching"*
- **동작 변화**: Kakao 서버가 카톡 앱 전환 대신 웹 인증 페이지만 표시 → -1002 원천 차단
- **검증**: TypeScript 0 에러, esbuild 번들 통과
- **영향 범위**: Google/Apple/로그아웃 모두 안 건드림
- **재빌드**: 불필요 (서버만, Replit 배포)

### ⏸️ 보류 (재빌드 사이클 회피)
