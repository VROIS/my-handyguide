# 최종 사용자 시나리오 — "여행의 모든 순간에 함께하는 자비스"

> 목적: 초행 해외여행자가 대화형으로 모든 어려움을 도움받는 AI 동반자
> 작성일: 2026-04-05
> 상태: RN 독립 모듈 개발 중 (main-input-screen-rn-v1.0)

---

## 사용자 = 초행 해외여행자

```
처음 해외여행. 말도 안 통하고, 길도 모르고, 뭘 먹어야 할지도 모름.
가방 들고, 사진 찍고, 도둑 조심하느라 폰 볼 여유 없음.
이어폰만 꽂으면 옆에서 걸어가는 AI 가이드가 있다면?
```

## 핵심 원칙

**음성 중심 UX — 화면은 보조적**
```
기존 앱: 눈 → 손 → 화면 터치 → 결과 읽기
우리 앱: 귀 → 입 → 음성 명령 → 음성 응답
```

**자비스가 하는 것:**
- 카메라로 보이는 것을 실시간 설명
- 사용자가 말하면 바로 도와줌
- 위험하면 알아서 도와줌 (SOS)
- 특별한 순간을 기억해줌 (보관함)
- 메뉴판을 읽어줌 (번역 + 알레르기 경고)
- 길을 잃으면 안내

**핵심: 버튼이 목적이 아님. 대화 하나로 모든 것이 연결됨.**
Gemma 4의 Tool Calling이 자동 판단 — 사용자는 그냥 말하기만 하면 됨.

---

## 5개 버튼 구성

| 버튼 | 아이콘 | 핵심 동작 |
|------|--------|----------|
| 라이브 | signal | Always-Listening + 카메라 AI (자비스 ON/OFF) |
| 촬영 | camera | 즉시 캡처 → AI 분석 → 음성 응답 |
| 업로드 | image | 갤러리 → AI 분석 |
| 여행비서 | globe | 통역/환율/SOS/메뉴판/교통/위치공유 |
| 보관함 | archive | 저장된 순간들 다시 보기 |

## 사용자 시나리오

### 시나리오 A: 공항 도착

```
[앱 첫 실행]
→ Gemma 4 E2B 모델 다운로드 중 (~2.4GB, Wi-Fi)
→ 기능 소개 슬라이드 표시
→ "준비됐어요!" 음성

사용자: (이어폰 꽂고 라이브 버튼)
자비스: "안녕하세요! 여행 가이드입니다. 무엇이든 물어보세요."

사용자: "여기 어디야?"
→ GPS → 역지오코딩
→ "인천국제공항 제1터미널에 계세요."

사용자: "환전 어디서 해?"
→ Gemma Tool Calling → getLocation() + 사전학습 지식
→ "1층 도착 로비에 환전소가 있어요. 현재 1달러 = 1,350원이에요."

사용자: (카메라로 간판 비춤)
→ 카메라 프레임 → Gemma 이미지 분석
→ "이 간판은 공항 리무진 버스 안내판이에요. 서울역까지 16,000원..."
```

### 시나리오 B: 관광지

```
사용자: (에펠탑을 카메라로 비추며) "이거 뭐야?"
→ 카메라 프레임 + 음성 → Gemma 멀티모달
→ "에펠탑이에요! 1889년 만국박람회를 위해 지어졌고..." (TTS)

사용자: "사진 찍어줘"
→ 촬영 버튼 → 즉시 캡처 → AI 설명 자동 생성
→ 상세 화면에서 저장 → 보관함에 이미지+설명 저장 (로컬+서버)

사용자: "에펠탑 어떻게 가?"
→ GPS 기반 → "현재 위치에서 도보 15분, 지하철 6호선 Bir-Hakeim역..."
```

### 시나리오 C: 식당

```
사용자: (메뉴판을 카메라로 비추며) "이거 뭐야?"
→ 카메라 → Gemma 이미지 분석 (멀티모달)
→ "프랑스어 메뉴판이네요. 첫 번째는 오리 콩피..."
→ Tool Calling → translateMenu() → 알레르기 경고 자동 포함
→ "⚠️ 3번 메뉴에 견과류가 포함되어 있어요."

사용자: "이거 얼마야?"
→ Tool Calling → getExchangeRate("EUR", "KRW")
→ "25유로는 약 36,000원이에요."

사용자: "이 요리 추천해줘"
→ Gemma 사전학습 → "오리 콩피는 프랑스 남서부 전통 요리로..."
```

### 시나리오 D: 긴급 상황

```
사용자: "위험해요" / "도와주세요" / "SOS"
→ Gemma 키워드 감지 → 자동 SOS 모드
→ GPS 위치 확보 → 역지오코딩
→ "가장 가까운 경찰서는 200m 앞이에요. 전화 연결할까요?"
→ 원터치 긴급 전화 (국가별 자동: 112/911/110)
→ 동시에 가족에게 위치 공유 (SMS/카카오톡)
   "🆘 긴급! 현재 위치: 파리 샹젤리제 거리\n지도: https://maps.google.com/..."

사용자: "길을 잘못 들었어요"
→ GPS → "50m 뒤로 돌아가서 왼쪽으로 가세요."

사용자: "지금 어디야?" (여정 공유 중 가족이 물어볼 때)
→ GPS → "현재 루브르 박물관 근처에 있어요."
```

### 시나리오 E: 보관함 (여행 후)

```
여행 끝나고 집에서 → 보관함 버튼
→ 저장한 모든 순간들:
  - 에펠탑 사진 + AI 설명
  - 메뉴판 번역
  - 여행 경로 기록
  - 음성으로 질문했던 대화들

→ 로컬(AsyncStorage)에도 있고 서버(PostgreSQL)에도 있어서
  어디서든 접근 가능 (웹 보관함과 동일 DB)

→ 공유 가능: 친구에게 여행 기록 공유 링크 전송
```

## 시나리오 F: 모델 다운로드 온보딩 (가장 중요)

### 첫 실행 — 소개영상 + 백그라운드 다운로드 (메인 흐름)

**기존 기능설명 페이지를 대체 → 28초 소개영상을 반드시 감상해야 메인 진입**
**영상 감상 동안 백그라운드에서 모델 다운로드+설치 → 사용자는 "대기"를 느끼지 못함**

```
앱 설치 → 첫 실행
  ↓
[언어 선택] (기존 온보딩 — 이미 구현됨)
  ↓
[소개영상 시작] ← 기존 기능설명 페이지 대체 (Remotion 등으로 제작)
  │
  │  영상 내용 (~28초):
  │  0s  "여행의 모든 순간을 내 손안에"
  │  5s  이어폰 꽂는 장면 → "이어폰만 꽂으면 AI 가이드가 함께해요"
  │  10s 카메라로 건물 비추기 → "보이는 모든 것을 실시간 설명"
  │  15s 말하는 장면 → "말만 하면 통역, 환율, 길 안내"
  │  20s SOS 장면 → "위험할 때 자동으로 도와드려요"
  │  25s "인터넷 없이도, 어디서든" → 오프라인 아이콘
  │  28s "준비됐어요!" → 페이드아웃
  │
  │  [동시에 백그라운드]
  │  0s  HuggingFace → 모델 다운로드 시작 (2.4GB)
  │  ~28s 다운로드 완료 (Wi-Fi 기준 충분)
  │       + LiteRT-LM 엔진 초기화
  │
  ↓
[영상 끝 + 다운로드 완료] → 둘 다 충족 시 자동 전환
  ↓
메인 화면 (카메라 + 5개 버튼) → "준비됐어요!" 음성
  ↓
자비스 시작
```

**사전 안내 (랜딩페이지 언어선택 하단):**
```
[언어 선택 화면]

  🇰🇷 한국어
  🇺🇸 English
  🇯🇵 日本語
  ...

  ─────────────────────────
  ☁️ 최초 실행 시 AI 가이드 설치를 위해
     약 30초와 3GB의 저장공간이 필요합니다.
     Wi-Fi 환경에서 시작해주세요.
  ─────────────────────────

  [시작하기]
```

→ 사용자가 **미리 알고 시작** — 30초 대기에 불만 없음
→ 언어 선택 후 [시작하기] → 소개영상 + 백그라운드 다운로드
→ 이미 마음의 준비가 되어있으므로 스킵 욕구 감소
→ 저장공간 부족 시 미리 정리할 수 있음

**영상이 끝났는데 다운로드 미완료 시:**
```
→ 하단에 작은 프로그레스 바만 표시
→ "거의 다 됐어요..." (남은 시간 표시)
→ 완료 즉시 자동 전환
```

**Wi-Fi 없어서 다운로드 불가 시:**
```
→ 영상은 정상 재생 (영상은 앱에 내장)
→ 영상 끝 → 온라인 모드로 메인 진입 (Gemini API)
→ 이후 Wi-Fi 감지 시 백그라운드 자동 다운로드 (아래 참조)
```

### Wi-Fi 없이 시작한 경우 (온라인 폴백 → 반드시 온보딩)
```
앱 설치 → 첫 실행 → Wi-Fi 없음
→ "지금은 온라인 모드로 시작합니다."
→ 모든 기능 정상 동작 (Gemini API 사용 — 사용자는 차이 느끼기 어려움)

[이후 자연스러운 온보딩 유도]

상황 1: Wi-Fi 감지 시 부드러운 알림
→ 상단 배너 (비침입적):
  "Wi-Fi에 연결되었어요. AI를 다운로드하면 인터넷 없이도 사용할 수 있어요!"
  [다운로드] [나중에]

상황 2: 오프라인 진입 시 자연스러운 유도
→ 비행기 모드 또는 해외 데이터 꺼짐 감지
→ "인터넷이 없어 기능이 제한됩니다."
→ "AI를 다운로드하면 오프라인에서도 모든 기능을 사용할 수 있어요!"
→ 장점 강조:
  ✅ 인터넷 없어도 통역/가이드/SOS
  ✅ 데이터 요금 0원 (해외 로밍 걱정 없음)
  ✅ 더 빠른 응답 (서버 왕복 없음)
  ✅ 개인정보 보호 (데이터가 폰 밖으로 안 나감)
→ [Wi-Fi 연결 후 다운로드] 버튼

상황 3: 여행 전 준비 화면
→ 설정 → "오프라인 AI 준비" 메뉴
→ "여행 전에 Wi-Fi에서 다운로드하세요 (약 2.4GB, 2~3분)"
→ 장점 카드 표시:
  "비행기 안에서도 통역 가능"
  "해외에서 데이터 없이 가이드"
  "응답 속도 3배 빠름"
→ [지금 다운로드]

상황 4: 사용 중 자연스러운 전환
→ Wi-Fi 연결 + 앱 사용 중 → 백그라운드 자동 다운로드 시작
→ 완료 → 부드러운 토스트: "오프라인 AI 준비 완료! 이제 어디서든 함께해요."
→ 자동으로 온디바이스 전환 (사용자 조작 불필요)
```

### 핵심 원칙
- **절대 강제하지 않음** — 온라인 모드로도 100% 사용 가능
- **장점을 체감시킴** — 오프라인/속도/비용/프라이버시
- **최적 타이밍에 유도** — Wi-Fi 감지, 오프라인 진입, 여행 전
- **백그라운드 다운로드** — 사용 중단 없이 자동 완료
- **전환 무감지** — 다운로드 완료 후 자동으로 온디바이스 전환

## AI 엔진 — Gemma 4 E2B 온디바이스

| 항목 | 스펙 |
|------|------|
| 모델 | Gemma 4 E2B (5.1B 파라미터, 실효 2.3B) |
| 크기 | ~2.4GB (.litertlm, int4 양자화) |
| 멀티모달 | 이미지 + 비디오 + 오디오 |
| 오디오 | USM Conformer — STT 없이 음성 직접 이해 |
| 통역 | 140개국어 오프라인 |
| 라이선스 | Apache 2.0 (완전 무료 상업 사용) |
| 프레임워크 | LiteRT-LM (Google 공식) |
| 타겟 기기 | Galaxy A35 (6-8GB RAM) ✅, iPhone 12 (4GB) ✅ |

**오프라인 가능:**
- 카메라 → 이미지 분석 → AI 해설
- 음성 대화 (네이티브 오디오)
- 140개국어 통역
- 여행지 역사/문화/코스 추천

**온라인 필요 (실시간 정보):**
- 실시간 환율
- 오늘 날씨
- 최신 맛집/웨이팅 정보
